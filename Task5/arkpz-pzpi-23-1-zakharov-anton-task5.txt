Харків 2025 

1.	Мета роботи

Розгорнути реалізовану IoT-систему моніторингу стану тварин FarmEnviroControl, перевірити її функціонування в середовищі симуляції Wokwi та локальному серверному середовищі, а також продемонструвати роботу бізнес-логіки обробки показників стану тварин, механізмів сигналізації критичних станів і взаємодію IoT-клієнта з серверною частиною системи через REST API.
2.	Хід роботи
2.1.	Опис розгорнутої програмної системи
IoT-клієнт (ESP32, Wokwi):
-	Мова програмування: C++ у середовищі Arduino IDE.
-	Апаратна платформа: ESP32 (емуляція в Wokwi).
-	Сенсорні дані: Емульовані значення температури тіла тварини та пульсу.
-	Індикація стану:
-	Світлодіод (LED) для сигналізації нормального та критичного стану.
-	Бузер для звукового сповіщення у разі критичних показників.
-	Мережеве з’єднання: Wi-Fi модуль ESP32 для підключення до локальної мережі.
Функціональність:
-	Генерація показників температури та пульсу тварини.
-	Аналіз даних відповідно до заданих критичних порогів.
-	Визначення загального стану тварини (GOOD, NORMAL, BAD).
-	Активація світлової та звукової сигналізації при небезпечних станах.
-	Передача даних на сервер через HTTP REST API.
-	Логування роботи пристрою через серійний монітор.
Серверна частина:
-	Мова програмування: JavaScript.
-	Платформа: Node.js.
-	Фреймворк: Express.js для побудови REST API.
-	База даних: MongoDB з використанням бібліотеки Mongoose.
Функціональність:
-	Прийом HTTP-запитів від IoT-клієнта.
-	Обробка JSON-даних із показниками стану тварини.
-	Збереження даних у базі даних.
-	Підготовка даних для подальшого аналізу та відображення у клієнтських додатках.
Інтеграція IoT та сервера:
-	Протокол обміну: HTTP (REST API).
-	Формат даних: JSON.
-	Тип переданих даних: температура, пульс, загальний стан тварини, часові мітки.
2.2.	Опис розгортання
IoT-клієнт (ESP32):
-	Ініціалізація ESP32 та налаштування портів введення/виведення для світлодіода і бузера.
-	Підключення до Wi-Fi мережі та перевірка стабільності з’єднання.
-	Генерація емульованих значень температури та пульсу тварини.
-	Аналіз отриманих значень та визначення стану тварини:
-	Good — нормальні показники.
-	Normal — прикордонні значення, що потребують спостереження.
-	Bad — критичні значення.
-	Активація світлової та звукової індикації у разі небезпечного стану.
-	Формування інформаційних повідомлень та логування результатів у серійний монітор.
-	Передача даних на сервер через HTTP-запит.
Серверна частина (Node.js + MongoDB):
-	Встановлення необхідних залежностей за допомогою npm.
-	Налаштування параметрів підключення до бази даних MongoDB.
-	Запуск серверного застосунку та перевірка доступності REST API.
-	Прийом даних від IoT-клієнта.
-	Збереження показників стану тварини у базі даних.
-	Перевірка коректності збережених даних через API-запити.
Інтеграція IoT та сервера:
-	ESP32 встановлює Wi-Fi з’єднання.
-	Дані про стан тварини надсилаються HTTP-запитами на сервер.
-	Сервер приймає дані, обробляє їх та зберігає у базі даних.
-	Дані доступні для подальшого використання у системі моніторингу ферми.
Використані інструменти та технології:
-	ESP32 (Wokwi) — IoT-клієнт.
-	Arduino IDE — розробка програмного забезпечення.
-	Node.js, Express.js — серверна частина.
-	MongoDB, Mongoose — збереження даних.
-	HTTP REST API — обмін даними.
-	JSON — формат передачі даних.
-	GitHub — керування версіями коду.



3.	Висновки
У ході виконання лабораторної роботи було успішно розгорнуто та протестовано IoT-систему FarmEnviroControl для моніторингу стану тварин у фермерських господарствах. Реалізований IoT-клієнт на базі мікроконтролера ESP32 стабільно працює в середовищі симуляції Wokwi, коректно генерує показники температури та пульсу тварин, виконує аналіз отриманих даних і визначає загальний стан тварини відповідно до заданих порогових значень.
Реалізована бізнес-логіка дозволяє своєчасно виявляти критичні стани та активувати світлову і звукову сигналізацію, що підвищує надійність системи та забезпечує оперативне реагування на небезпечні ситуації. Передача даних на серверну частину через REST API підтвердила стабільну взаємодію між IoT-клієнтом і сервером.
Серверна частина системи, розроблена на платформі Node.js з використанням Express.js та MongoDB, успішно приймає та зберігає дані про стан тварин, забезпечуючи можливість їх подальшого аналізу та використання у системах управління фермою. Проведене тестування підтвердило коректність роботи API та збереження інформації у базі даних.
Розроблена система є масштабованою та готовою до подальшого розширення, зокрема інтеграції з веб-інтерфейсом користувача, автоматичними сповіщеннями та аналітичними модулями. FarmEnviroControl може бути використана як основа для сучасних систем моніторингу стану тварин на фермах, забезпечуючи підвищення ефективності управління, своєчасне виявлення проблем зі здоров’ям тварин та загальну автоматизацію процесів контролю.
